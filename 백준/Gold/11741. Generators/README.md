### 문제 설명

<p>Little Roman is studying linear congruential generators &mdash; one of the oldest and best known pseudorandom number generator algorithms. Linear congruential generator (LCG) starts with a non-negative integer number x<sub>0</sub> also known as seed and produces an infinite sequence of non-negative integer numbers x<sub>i</sub> (0 &le; x<sub>i</sub> &lt; c) which are given by the following recurrence relation:</p>

<p style="text-align:center">x<sub>i+1</sub> = (ax<sub>i</sub> + b) mod c</p>

<p>here a, b, and c are non-negative integer numbers and 0 &le; x<sub>0</sub> &lt; c.</p>

<p>Roman is curious about relations between sequences generated by different LCGs. In particular, he has n different LCGs with parameters a<sup>(j)</sup>, b<sup>(j)</sup>, and c<sup>(j)</sup> for 1 &le; j &le; n, where the j-th LCG is generating a sequence x<sub>i</sub><sup>(j)</sup>. He wants to pick one number from each of the sequences generated by each LCG so that the sum of the numbers is the maximum one, but is not divisible by the given integer number k.</p>

<p>Formally, Roman wants to find integer numbers t<sub>j</sub> &ge; 0 for 1 &le; j &le; n to maximize s = &Sigma;<sup>n</sup><sub>j=1</sub> x<sub>t<sub>j</sub></sub><sup>(j)</sup> &nbsp;subject to constraint that s mod k &ne; 0.</p>

### 입력

<p>The first line of the input file contains two integer numbers n and k (1 &le; n &le; 10 000, 1 &le; k &le; 10<sup>9</sup>). The following n lines describe LCGs. Each line contains four integer numbers x<sub>0</sub><sup>(j)</sup>, a<sup>(j)</sup>, b<sup>(j)</sup>, and c<sup>(j)</sup> (0 &le; a<sup>(j)</sup>, b<sup>(j)</sup> &le; 1000, 0 &le; x<sub>0</sub><sup>(j)</sup>&nbsp;&lt; c<sup>(j)</sup> &le; 1000).</p>

### 출력

<p>If Roman&rsquo;s problem has a solution, then write on the first line of the output file a single integer s &mdash; the maximum sum not divisible by k, followed on the next line by n integer numbers t<sub>j</sub> (0 &le; t<sub>j</sub> &le; 10<sup>9</sup>) specifying some solution with this sum.</p>

<p>Otherwise, write to the output file a single line with the number &minus;1.</p>

### 힌트

<p>In the first example, one LCG is generating a sequence 1, 2, 3, 4, 5, 0, 1, 2, . . ., while the other LCG a sequence 2, 3, 2, 3, 2, . . ..</p>

<p>In the second example, one LCG is generating a sequence 0, 2, 0, 2, 0, . . ., while the other LCG a sequence 2, 4, 2, 4, 2, . . ..</p>
