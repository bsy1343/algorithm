# [Gold I] 저항 - 1483

[문제 링크](https://www.acmicpc.net/problem/1483)

### 성능 요약

시간 제한: 2 초, 메모리 제한: 128 MB

### 통계

제출: 88, 정답: 12, 맞힌 사람: 4, 정답 비율: 19.048%

### 분류

수학, 다이나믹 프로그래밍, 백트래킹

### 문제 설명

<p>저항 R<sub>1</sub>, R<sub>2</sub>, R<sub>3</sub> ,... R<sub>n</sub>을 직렬로 연결 하면, (R<sub>1</sub>+R<sub>2</sub>+R<sub>3</sub>+...+Rn)이 되고, 병렬로 연결 하면, 1/((1/R<sub>1</sub>)+(1/R<sub>2</sub>)+(1/R<sub>3</sub>)+...+(1/R<sub>n</sub>))이 된다.</p>

<p>두 정수 a와 b가 주어진다. 크기가 1과 2인 저항을 매우 많이 가지고 있을때, 저항을 몇 개를 연결해야 a/b가 되는지 출력하는 프로그램을 작성하시오.</p>

### 입력

<p>첫째 줄에 a와 b가 주어진다. a와 b는 50,000보다 작거나 같은 자연수이다.</p>

### 출력

<p>첫째 줄에 필요한 저항의 최소 개수를 출력한다. 만약 필요한 저항의 수가 16개보다 크다면 -1을 출력한다.</p>

### 힌트

<p>예제 1의 경우 1과 2를 직렬 연결한 것을 2와 병렬 연결 하면 된다.</p>
