### 문제 설명

<p>Как Вы уже могли заметить, в жюри Russian Code Cup любят обратные задачи. Вот еще одна!</p>

<p>Система непересекающихся множеств &mdash; это структура данных, которая позволят хранить набор элементов, представленных в виде набора непересекающихся множеств. Она позволяет выполнять две операции:</p>

<ul>
	<li>union(<i>a</i>,&nbsp;<i>b</i>) &mdash; объединяет два множества, содержащие элементы&nbsp;<i>a</i>&nbsp;и&nbsp;<i>b</i>.</li>
	<li>find(<i>a</i>) &mdash; возвращает некоторого представителя множества, в котором содержится&nbsp;<i>a</i>. find(<i>a</i>) = find(<i>b</i>) тогда и только тогда, когда&nbsp;<i>a</i>&nbsp;и&nbsp;<i>b</i>&nbsp;содержатся в одном множестве.</li>
</ul>

<p>Популярным подходом к реализации данной структуры данных является реализация в виде леса корневых деревьев. Каждое множество представляется в виде корневого дерева, представителем которого является корень дерева. Родителя вершины&nbsp;<i>i</i>&nbsp;обозначим&nbsp;<i>parent</i>[<i>i</i>] (для корня дерева положим&nbsp;<i>parent</i>[<i>i</i>] =&nbsp;<i>i</i>). Изначально, каждый элемент находится в своем множестве и&nbsp;<i>parent</i>[<i>i</i>] =&nbsp;<i>i</i>&nbsp;для всех&nbsp;<i>i</i>. Операция union в таком представлении реализуется, как подвешивание одного из корней дерева к другому. Операция find реализуется, как простой проход по ссылкам к родителю, пока не достигнут корень дерева. Для простоты будем считать, что аргументы процедуры union &mdash; корни различных деревьев.</p>

<p>Также в этой задаче используется ранговая эвристика: введем дополнительную величину&nbsp;<i>rank</i>[<i>i</i>], которая изначально равна нулю для всех&nbsp;<i>i</i>. Операция union(<i>a</i>,&nbsp;<i>b</i>) работает следующим образом: если&nbsp;<i>rank</i>[<i>a</i>] =&nbsp;<i>rank</i>[<i>b</i>], то&nbsp;<i>rank</i>[<i>a</i>] увеличивается на единицу. После этого вершина с меньшим&nbsp;<i>rank</i>&nbsp;подвешивается к вершине с большим&nbsp;<i>rank</i>.</p>

<p>Приведем псевдокод процедур union и find.</p>

<pre>
union(a, b)
  if rank[a] == rank[b] then
    rank[a] = rank[a] + 1
  if rank[a] &gt; rank[b] then
    parent[b] = a
  else
    parent[a] = b

find(a)
  while parent[a] != a do
    a = parent[a]
  return a
</pre>

<p>Вам дан массив&nbsp;<i>parent</i>[<i>i</i>]. Существует ли такая последовательность операций&nbsp;<i>union&nbsp;</i>, что после ее выполнения массив&nbsp;<i>parent&nbsp;</i>станет равен заданному?</p>

### 입력

<p>Первая строка содержит целое число&nbsp;<i>n</i>&nbsp;(1&nbsp;&le;&nbsp;<i>n</i>&nbsp;&le;&nbsp;10<sup>4</sup>). Вторая строка содержит&nbsp;<i>n</i>&nbsp;целых чисел&nbsp;<i>parent</i>[1], ...,&nbsp;<i>parent</i>[<i>n</i>] (1&nbsp;&le;&nbsp;<i>parent</i>[<i>i</i>]&nbsp;&le;&nbsp;<i>n</i>&nbsp;для всех&nbsp;<i>i</i>&nbsp;от 1 до&nbsp;<i>n</i>).</p>

### 출력

<p>Если требуемой последовательности не существует, выведите -1. Иначе в первой строке выведите целое число&nbsp;<i>k</i>&nbsp;&ge; 0 &mdash; количество операций. Затем выведите&nbsp;<i>k</i>&nbsp;пар чисел&nbsp;<i>a<sub>i</sub></i>&nbsp;<i>b<sub>i</sub></i>&nbsp;(1&nbsp;&le;&nbsp;<i>a<sub>i</sub></i>,&nbsp;<i>b<sub>i</sub></i>&nbsp;&le;&nbsp;n) &mdash; аргументы&nbsp;<i>i</i>-й операции union. К моменту выполнения&nbsp;<i>i</i>-й операции&nbsp;<i>a<sub>i</sub></i>&nbsp;и&nbsp;<i>b<sub>i</sub></i>&nbsp;должны быть корнями различных деревьев.</p>
