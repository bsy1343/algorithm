# [Gold III] 좋은 배열 만들기 - 2746

[문제 링크](https://www.acmicpc.net/problem/2746)

### 성능 요약

시간 제한: 1 초, 메모리 제한: 512 MB

### 통계

제출: 1141, 정답: 210, 맞힌 사람: 148, 정답 비율: 20.728%

### 분류

정렬, 이분 탐색, 많은 조건 분기

### 문제 설명

<p>배열의 어떤 원소가 다른 모든 원소의 합과 같을 때, 이러한 배열을 &quot;좋은 배열&quot;이라고 하자. 예를 들어 배열 $a = [1,\ 2,\ 3,\ 6]$은 $1+2+3=6$이기 때문에 &quot;좋은 배열&quot;이다.</p>

<p>길이가 $N$인 배열 $A$가 주어지면, 정확히 2개의 원소를 제거해서 &quot;좋은 배열&quot;을 만드는 방법의 수를 구해보자.</p>

<p>예를 들어 $a = [1,\ 3,\ 6,\ 8,\ 9]$의 경우, 첫째 원소와 넷째 원소를 제거하면 $3+6=9$를 만족하고, 둘째 원소와 셋째 원소를 제거하면 $1+8=9$를 만족하므로 정답은 2이다.</p>

### 입력

<p>첫째 줄에 배열의 길이 $N$이 주어진다.</p>

<p>둘째 줄에 배열의 원소 $A_1, A_2, \cdots, A_N$이 공백으로 구분되어 주어진다.</p>

### 출력

<p>정확히 2개의 원소를 제거해서 &quot;좋은 배열&quot;을 만들 때, 원소를 제거하는 방법의 수를 출력한다.</p>

### 제한

<ul>
	<li>$5 \leq N \leq 500\,000$</li>
	<li>$1 \leq A_i \leq 1\,000\,000$, $A_i$는 정수</li>
	<li>32bit 정수를 사용하는 경우 overflow가 발생할 수 있음에 주의하자.</li>
</ul>
