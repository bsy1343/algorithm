### 문제 설명

<p>Источник, пожелавший остаться неизвестным, прислал Дэдпулу зашифрованное сообщение, в котором сообщались возможные координаты базы противника. Возможные координаты представляли собой отрезок натуральных чисел от $x_1$ до $x_2$ включительно. Шифровалось сообщение довольно простым способом: берется последовательность десятичных записей всех натуральных чисел, записанных подряд без пропусков ($1234567891011121314...$) и сообщается ее подотрезок $[l, r]$, отвечающий за числа с $x_1$ по $x_2$ включительно. Например, если $x_1=1, x_2=6$, зашифрованное сообщение будет выглядеть так: $[1, 6]$. А если $x_1=11, x_2=14$, то так: $[12, 19]$.</p>

<p>Однако, получив сообщение, Дэдпул сразу понял, что оно дошло не совсем правильно, возможно, кто-то его испортил по пути. Делать нечего, поэтому теперь по данным числам $l$, $r$ он хочет понять хотя бы примерные координаты базы: найти все натуральные числа, которые полностью входят в отрезок шифровки.</p>

### 입력

<p>В первой строке входного файла записано два числа $l$, $r$ ($1 \le l \le r \le 10^{18}$) --- шифровка, полученная Дэдпулом.</p>

### 출력

<p>В единственной строке выходного файла выведите количество натуральных чисел, полностью содержащихся в отрезке $[l, r]$ строки &lt;&lt;<code>1234567891011...</code>&gt;&gt;, то есть координаты, в которых может находиться база противника. Для лучшего понимания условия смотрите примеры.</p>

### 힌트

<p>В первом тесте подходят все натуральные числа от 1 до 9.</p>

<p>Во втором тесте подходят только числа 8, 9, 10, 11, 12, 13. У числа 14 в данном отрезке содержится только первая его цифра, поэтому оно не учитывается.</p>
