### 문제 설명

<p>게임 개발자가 꿈인 영우는 컴퓨터게임설계 수업의 과제로 <a href="/problem/20211">RPG 게임</a>을 제출했다. 잇창명은 이 게임을 보고 감명을 받아 게임 개발자의 꿈을 가지게 되었지만, 현실의 높은 벽에 부딪혀 게임 광고 개발자로 만족할 수밖에 없었다.</p>

<p>잇창명이 만든 인터랙티브 광고는 $N$행 $M$열의 격자 위에서 진행된다. 가장 왼쪽 위 격자점의 좌표는 $(1,1)$, 가장 오른쪽 아래 격자점의 좌표는 $(N,M)$이다.</p>

<p>격자 위에는 여러 개의 화살표가 배치되어 있다. 화살표는 한 격자점에서 시작해 다른 격자점에서 끝나는 길이 $2$ 이상의 격자점의 나열이다. 화살표의 마지막 두 격자점이 이루는 방향을 그 화살표가 바라보는 방향이라고 한다. 화살표가 자신과 겹치는 경우는 없으며, 모든 격자점은 최대 한 개의 화살표에만 속해 있다.</p>

<p>플레이어가 화살표를 선택하면 그 화살표는 바라보는 방향으로 이동한다. 화살표의 이동은 다음 순서대로 이루어진다.</p>

<ol>
<li>화살표의 마지막 두 격자점을 순서대로 $(r,c)$와 $(r+\Delta_r,c+\Delta_c)$라고 할 때, $(r+2\cdot\Delta_r,c+2\cdot\Delta_c)$를 화살표의 마지막에 추가한다.</li>
<li>화살표의 첫 격자점을 제거한다.</li>
<li>새로 추가하는 격자점이 격자를 벗어나기 전까지 위의 과정을 반복한다.</li>
</ol>

<p>이 과정을 수행하는 중에 자신을 포함한 어떤 화살표와도 겹치지 않을 경우 플레이어는 그 화살표를 제거할 수 있다.</p>

<p>화살표가 이동하면서 자신을 포함한 어떤 화살표와 겹치는 경우 이동을 멈추고 원래 위치로 돌아온다. 화살표가 이동할 때 격자점이 추가된 뒤에 처음 격자점을 제거하므로 자신의 시작점에 겹치는 화살표는 제거할 수 없다.</p>

<p>화살표가 이동 중일 때는 다른 화살표를 선택할 수 없다.</p>

<p>광고 플랫폼은 인터랙티브 광고를 평가할 때 실제로 클리어할 수 있는지 확인한다. 주어진 격자에서 모든 화살표를 제거할 수 있는지 확인하는 프로그램을 작성하여라.</p>

<p>혹시 문제가 설명하는 게임을 어디서 많이 본 것 같다면 잇창명에게 감사를 표하도록 하자.</p>

### 입력

<p>첫 번째 줄에 격자의 크기 $N$, $M$, 화살표의 개수 $K$가 공백으로 구분되어 주어진다. $(2\le N,M\le 200$; $1\le K\le \left\lfloor \frac{NM}{2} \right\rfloor)$</p>

<p>두 번째 줄부터 $K$개의 줄에 걸쳐 화살표의 정보가 주어진다. 그중 $i$번째 줄은 $i$번째 화살표에 대한 정보 $r_i$, $c_i$, $l_i$, 길이가 $l_i$인 문자열 $D_i$가 공백으로 구분되어 주어진다. $(1\le r_i\le N$; $1\le c_i\le M$; $1\le l_i \lt NM$; $D_{i,j}\in\{$<span style="color:#e74c3c;"><code>U</code></span>$, $<span style="color:#e74c3c;"><code>D</code></span>$, $<span style="color:#e74c3c;"><code>L</code></span>$, $<span style="color:#e74c3c;"><code>R</code></span>$\})$</p>

<p>$i$번째 화살표의 길이는 $l_i+1$이며 $D_{i,j}$는 $j$번째 격자점에서 $j+1$번째 격자점을 바라보는 방향이다. 문자열의 문자는 <span style="color:#e74c3c;"><code>U</code></span>는 위, <span style="color:#e74c3c;"><code>D</code></span>는 아래, <span style="color:#e74c3c;"><code>L</code></span>은 왼쪽, <span style="color:#e74c3c;"><code>R</code></span>는 오른쪽을 의미한다. 행이 증가하는 방향이 아래, 열이 증가하는 방향이 오른쪽이다. 예를 들어 예제 입력 2의 2번째 화살표는 다음과 같이 이루어져 있다.</p>

<p style="text-align: center;"><img alt="" src="%EB%B0%B1%EC%A4%80/Gold/35462.%E2%80%85%EA%B2%8C%EC%9E%84%E2%80%85%EA%B4%91%EA%B3%A0%E2%80%85%EA%B0%9C%EB%B0%9C%EC%9E%90%E2%80%85%EC%9E%87%EC%B0%BD%EB%AA%85/b45355c0.jpg" data-original-src="https://boja.mercury.kr/assets/problem/35462-1.jpg" onerror="this.onerror=null;this.src=this.dataset.originalSrc" style="display: block; margin: 0 auto; max-width: 100%; height: auto; width: 250px;"></p>

<p>화살표가 자신과 겹치거나 한 격자점이 둘 이상의 화살표에 포함되거나 화살표가 격자 밖으로 벗어나는 경우는 입력으로 주어지지 않는다.</p>

### 출력

<p>첫 번째 줄에 주어진 격자에서 모든 화살표를 제거할 수 있다면 <span style="color:#e74c3c;"><code>Yes</code></span>, 그렇지 않다면 <span style="color:#e74c3c;"><code>No</code></span>를 출력한다.</p>
