### 문제 설명

<p>A (formal) language is a set of strings. One way to define a particular langauge is using ordinary set notation. Alternatively, some form of grammar may be more convenient for representing large sets. The UW grammar in which we are interested has two parts:</p>

<ul>
	<li>An initial string</li>
	<li>A set of replacement rules of the form s1 -&gt; s2 where s1 and s2 are strings</li>
</ul>

<p>The language defined by this grammar is the set of all strings that can be generated by repeatedly replacing s1 by s2 within the initial string. For example, consider the grammar G consisting of the initial string</p>

<pre>&quot;AyB&quot;</pre>

<p>and the replacement rules</p>

<pre>{&quot;A&quot;-&gt;&quot;ab&quot;, &quot;Ay&quot;-&gt;&quot;cdy&quot;, &quot;B&quot;-&gt;&quot;w&quot;, &quot;B&quot;-&gt;&quot;x&quot;} .</pre>

<p>G generates the language</p>

<pre>L = {&quot;AyB&quot;, &quot;Ayw&quot;, &quot;Ayx&quot;, &quot;abyB&quot;, &quot;abyw&quot;, &quot;abyx&quot;, &quot;cdyB&quot;, &quot;cdyw&quot;, &quot;cdyx&quot;}</pre>

<p>Given a UW grammar G, compute how many different strings there are in the language generated by G.</p>

### 입력

<p>The first line of input contains the initial string. The second and subsequent lines contain the replacement rules, one per line, terminated by end-of-file. There are at most 100 replacement rules. Each input string contains between 0 and 10 upper and lower case letters, and is enclosed in quotes. There are no spaces in the input.</p>

### 출력

<p>Output consists of a single integer, the number of distinct strings in the language generated by G. If there are more than 1000 distinct strings, print &quot;Too many.&quot; instead.</p>
