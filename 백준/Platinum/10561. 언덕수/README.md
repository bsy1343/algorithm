# [Platinum IV] 언덕수 - 10561

[문제 링크](https://www.acmicpc.net/problem/10561)

### 성능 요약

시간 제한: 5 초, 메모리 제한: 256 MB

### 통계

제출: 133, 정답: 68, 맞힌 사람: 32, 정답 비율: 68.085%

### 분류

다이나믹 프로그래밍, 자릿수를 이용한 다이나믹 프로그래밍

### 문제 설명

<p>언덕수는 각 자릿수가 증가했다가 감소할 수는 있지만, 절대 감소했다가 증가하는 일은 없는 정수다.</p>

<ul>
	<li>12321은 언덕수다.</li>
	<li>101은 언덕수가 아니다.</li>
	<li>1111000001111은 언덕수가 아니다.</li>
</ul>

<p>정수 N이 주어지면, N이 언덕수라면 N보다 작은 언덕수의 개수를 출력하고, 언덕수가 아니라면 -1을 출력하라.</p>

### 입력

<p>첫째 줄에 테스트 케이스의 개수가 주어진다.</p>

<p>이어서 각 줄에 해당 테스트 케이스의 정수 N이 주어지며, N은 최대 70자리다. 답은 항상 64비트 long의 범위 안이 되도록 주어진다.</p>

### 출력

<p>각 테스트 케이스마다 한 줄에 걸쳐 답을 출력한다.</p>
