# [Platinum II] 행복한 수 - 11958

[문제 링크](https://www.acmicpc.net/problem/11958)

### 성능 요약

시간 제한: 0.5 초, 메모리 제한: 64 MB

### 통계

제출: 270, 정답: 46, 맞힌 사람: 33, 정답 비율: 17.010%

### 분류

수학, 정수론, 이분 탐색, 소수 판정, 런타임 전의 전처리, 에라토스테네스의 체

### 문제 설명

<p>준형이와 민형이가 수열 게임을 한다. 게임을 시작하기에 앞서, 민형이는 K, L, M 3개의 수를 뽑는다. 민형이가 수를 뽑았다면, 준형이는 K개의 연속한 자연수를 말해야 한다.</p>

<p>민형이는 준형이의 암산 실력을 키우기 위해서 룰을 하나 추가했다. 이 룰에 따르면,&nbsp;준형이가 K개의 수를 말하면&nbsp;그 중에서 &#39;행복한 수&#39;가 정확히 L개 있어야 한다. 행복한 수는 아래 두 조건 중 하나 이상을 만족시키는 수를 말한다.</p>

<ul>
	<li>수가 M 이하의 자연수인 경우</li>
	<li>수가 소수(약수가 정확히 2개인 수)인 경우</li>
</ul>

<p>준형이는 민형이의 질문에 빠른 대답을 하지 못하기 때문에 몰래 컴퓨터를 이용해서 답을 하려고 한다. 민형이의 질문에 답을 해주는 프로그램을 작성하여라.</p>

### 입력

<p>첫 번째 줄에는 테스트 케이스의 수 Q&nbsp;(1 &le; Q &le; 100 000)가 주어진다.</p>

<p>두 번째 줄부터 Q개의 줄에는 K, L, M의 값이 주어진다. (1 &le; K, M &le; 150, 0 &le; L &le; K)</p>

### 출력

<p>Q개의 줄에 준형이가 말해야 할 K개의 수들 중 가장 작은 수를 출력한다. 만약 답이 없거나 10 000 000보다 크다면, -1을 출력한다.</p>

<p>만약 답으로 가능한 수가 여러 개인 경우, 그중 아무거나 출력한다.</p>
