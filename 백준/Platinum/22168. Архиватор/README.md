### 문제 설명

<p>Вася придумал радикально новый способ сжатия текста. Он считает, что особенно хорошо его метод будет работать для языков с большим алфавитом, таких как китайский. По этой причине Вася представляет текст в виде последовательности кодов символов&nbsp;<i>a</i><sub>1</sub>,&nbsp;<i>a</i><sub>2</sub>, &hellip;,&nbsp;<i>a<sub>n</sub></i>.</p>

<p>Алгоритм сжатия разбит на фазы. На каждой фазе строится новая последовательность вдвое меньшей длины&nbsp;<i>b</i><sub>1</sub>,&nbsp;<i>b</i><sub>2</sub>, &hellip;,&nbsp;<i>b</i><sub>&lceil;<i>n</i>/2&rceil;</sub>. При этом&nbsp;<i>b<sub>i</sub></i>&nbsp;может равняться либо&nbsp;<i>a<sub>i</sub></i>, либо&nbsp;<i>a</i><sub><i>n&ndash;i+</i>1</sub>. Выбор оставляется на усмотрение архиватора. После этого исходная последовательность заменяется новой, и алгоритм переходит к следующей фазе. Процесс завершается, когда длина последовательности становится равной единице.</p>

<p>Вася заметил, что в случае неоднозначности (<i>a<sub>i</sub></i>&nbsp;&ne;&nbsp;<i>a</i><sub><i>n&ndash;i+</i>1</sub>) приходится сохранять дополнительные данные во избежание потери информации. Кроме того, от выбора символа на текущей фазе может зависеть наличие или отсутствие неоднозначностей в дальнейшем.</p>

<p>Для оценки эффективности алгоритма Вася решил назначать единицу штрафа за каждую неоднозначность, возникающую в процессе работы архиватора. К сожалению, Вася не смог придумать, каким образом нужно делать выбор при возникновении неоднозначностей, и обратился к вам за помощью. Он просит вас написать программу, которая вычислит минимальный возможный суммарный штраф при оптимальном разрешении неоднозначностей.</p>

### 입력

<p>Первая строка содержит целое число&nbsp;<i>n</i>&nbsp;(1 &le;&nbsp;<i>n</i>&nbsp;&le; 200000). Вторая строка содержит исходную последовательность &mdash;&nbsp;<i>n</i>&nbsp;целых чисел&nbsp;<i>a</i><sub>1</sub>,&nbsp;<i>a</i><sub>2</sub>, &hellip;,&nbsp;<i>a<sub>n</sub></i>&nbsp;(1&nbsp;&le;&nbsp;<i>a<sub>i</sub></i>&nbsp;&le;&nbsp;10<sup>9</sup>).</p>

### 출력

<p>Выведите целое число &mdash; минимальный суммарный штраф, который может быть достигнут при сжатии заданной последовательности по описанному алгоритму.</p>
