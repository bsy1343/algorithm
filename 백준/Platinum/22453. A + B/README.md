### 문제 설명

<p>プログラミングコンテストの合宿で出す問題のアイディアが出ずに困っていたイクタ君は、ある日友人に相談した。</p>

<p>イクタ君「こういうアルゴリズムを使わないと解けないような問題が出したいんですけど、なにかありませんかね？」</p>

<p>友人「それではこういうものを考えたらいいんじゃないですか」</p>

<p>このようにしてその友人は以下のような問題の原案となるアイデアを考えてくれた。</p>

<p>2進数$A,B$が与えられた時、以下の様なクエリを処理せよ。</p>

<ul>
	<li>
	<p>出力クエリ: max{$x$ を2進数で表したときの、1の数 | $A \leq x &lt; A+B$ }を出力</p>
	</li>
	<li>
	<p>A変更クエリ: $A$の最下位ビットから$i$ビット目(0-origin)を反転</p>
	</li>
	<li>
	<p>B変更クエリ: $B$の最下位ビットから$i$ビット目(0-origin)を反転</p>
	</li>
</ul>

<p>$i$ビット目というのが 0-origin で表されていることに注意せよ。 つまり、$A$の最下位ビットから0ビット目とは最下位ビットを表す。</p>

### 입력

<p>入力は以下の形式で与えられる。</p>

<blockquote>
<p>$N$ $A$ $B$</p>

<p>$Q_{1}$</p>

<p>...</p>

<p>$Q_{N}$</p>
</blockquote>

<p>1行目にクエリの数N,2進数A, Bが与えられる。</p>

<p>2～N+1行目には以下の様なクエリが与えられる。</p>

<ul>
	<li>
	<p>Q</p>
	</li>
	<li>
	<p>A $i$</p>
	</li>
	<li>
	<p>B $i$</p>
	</li>
</ul>

<p>Qが出力クエリを、A $i$,B $i$はそれぞれ$A$の最下位ビットから$i$ビット目、$B$の最下位ビットから$i$ビット目を反転させる変更クエリを表している。</p>

### 출력

<p>出力クエリごとに答えを1行に出力せよ。

### 제한

<p>入力中の各変数は以下の制約を満たす。</p>

<ul>
	<li>
	<p>$1 \leq N &lt; 300,000$</p>
	</li>
	<li>
	<p>$1 \leq |A|,|B| \leq 300,000$ ($|A|$は$A$の長さ)</p>
	</li>
	<li>
	<p>A $i$ というクエリでは $0 \leq i &lt; |A|$</p>
	</li>
	<li>
	<p>B $i$ というクエリでは $0 \leq i &lt; |B|$</p>
	</li>
	<li>
	<p>$B$が0になることはない</p>
	</li>
</ul>
