# [Platinum IV] 좋은 단순 경로 - 25123

[문제 링크](https://www.acmicpc.net/problem/25123)

### 성능 요약

시간 제한: 1.5 초  (하단 참고), 메모리 제한: 512 MB

### 통계

제출: 163, 정답: 58, 맞힌 사람: 50, 정답 비율: 34.247%

### 분류

다이나믹 프로그래밍, 트리, 트리에서의 다이나믹 프로그래밍

### 문제 설명

<p>노드가 $n$개인 트리 $T$가 있다. 노드는 $1$부터 $n$까지 번호가 붙어있고, 각 노드에는 영문 대문자 알파벳 (&#39;<code>A</code>&#39; - &#39;<code>Z</code>&#39;) 레이블이 붙어있다. Bob은 $T$에서 길이가 가장 긴 단순 경로를 찾는 문제에 관심이 많다. 단순 경로란, 그래프/트리의 같은 노드를 두 번 이상 방문하지 않는 경로를 말한다.</p>

<p>옆에서 지켜보던 Alice는 Bob에게 아래 정의에 따라 &quot;좋은 단순 경로&quot;를 찾아보라고 했다. 임의의 단순 경로 $P$에 대해, 만약 $P$에 속한 노드들의 레이블을 순서대로 나열하여 문자열을 만들었을 때 같은 알파벳이 연속으로 반복되지 않으면 $P$를 &quot;좋은 단순 경로&quot;라 한다.</p>

<p style="text-align: center;"><img alt="" src="%EB%B0%B1%EC%A4%80/Platinum/25123.%E2%80%85%EC%A2%8B%EC%9D%80%E2%80%85%EB%8B%A8%EC%88%9C%E2%80%85%EA%B2%BD%EB%A1%9C/583a50fd.png" data-original-src="https://upload.acmicpc.net/76f16b8d-1baf-4668-9b01-d5361bb331a5/-/preview/" onerror="this.onerror=null;this.src=this.dataset.originalSrc" style="height: 283px; width: 408px;" /></p>

<p>예를 들어 위의 트리는 $7$개의 노드를 포함하고 있다.</p>

<ul>
	<li>경로 $1 \to 5 \to 7$은 노드 $3$개를 포함하는 단순 경로이지만 노드들의 알파벳을 나열하면 &quot;<code>AAZ</code>&quot;가 되어 &#39;<code>A</code>&#39;가 연속으로 반복되므로 &quot;좋은 단순 경로&quot;가 아니다.</li>
	<li>경로 $3 \to 2 \to 1$은 노드 $3$개를 포함하는 좋은 단순 경로이다. 노드들의 알파벳을 나열하면 &quot;<code>AXA</code>&quot;가 되고 같은 알파벳이 여러 번 등장하지만 연속으로 반복되지 않는다.</li>
	<li>경로 $3 \to 2 \to 1 \to 4$와 경로 $4 \to 1 \to 2 \to 3$은 노드 $4$개를 포함하는 좋은 단순 경로이다. 단, 이 두 개의 경로는 같은 노드들을 포함하므로 같은 경로로 간주한다. (예제 입출력 참고)</li>
	<li>경로 $7$ 혹은 경로 $4$ 처럼 노드를 $1$개만 포함하는 좋은 단순 경로도 존재한다.</li>
	<li>이 트리에서 가장 긴 좋은 단순 경로의 길이는 $4$이며, 길이가 $4$인 좋은 단순 경로는 단 하나 존재한다.</li>
</ul>

<p>Alice와 Bob은 $T$에서 가장 긴 좋은 단순 경로의 길이가 무엇인지, 그리고 가장 긴 좋은 단순 경로의 개수가 몇 개인지 궁금해졌다. 둘을 도와 이 문제를 풀어보자.</p>

### 입력

<p>첫 줄에 테스트 케이스의 수 $T$가 주어진다</p>

<p>각 테스트 케이스는 세 줄에 걸쳐 주어진다. 첫 줄에 노드의 개수 $n$이 주어진다. 둘째 줄에 각 노드의 알파벳 레이블이 공백없이 길이 $n$인 문자열 형태로 주어진다. 셋째 줄에 각 노드의 부모 노드의 번호가 공백으로 구분되어 주어진다. 루트 노드의 부모는 $0$번으로 주어진다.</p>

### 출력

<p>각 테스트 케이스의 정답인 두 정수를 공백으로 구분하여 각 줄에 출력한다.</p>

<p>첫 번째 정수는 가장 긴 좋은 단순 경로의 길이 (노드의 수)를 나타내고, 두 번째 수는 서로 다른 좋은 단순 경로의 개수를 나타낸다.</p>

### 제한

<ul>
	<li>$1 &le; T &le; 10$</li>
	<li>각 노드의 레이블은 알파벳 대문자 (&#39;<code>A</code>&#39; - &#39;<code>Z</code>&#39;)이다.</li>
</ul>
