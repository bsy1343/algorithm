# [Platinum II] K-균형 잡힌 수 - 25336

[문제 링크](https://www.acmicpc.net/problem/25336)

### 성능 요약

시간 제한: 3 초, 메모리 제한: 1024 MB (추가 메모리 없음)

### 통계

제출: 54, 정답: 24, 맞힌 사람: 24, 정답 비율: 54.545%

### 분류

구현, 그리디 알고리즘

### 문제 설명

<p>우리가 주로 사용하는 10진법은 모든 수를 0부터 9까지의 숫자들의 조합으로 표기한다.</p>

<p>양의 정수&nbsp;$N$을 10진법으로 표기하기 위해 사용한 숫자들에 대해, 0으로 시작하지 않으면서 (가장 많이 사용한 숫자의 등장 횟수) - (가장 적게 사용한 숫자의 등장 횟수) $\leq K$이면&nbsp;$N$을 $K$-균형 잡힌 수라고 하자.</p>

<p>양의 정수&nbsp;$X, K$가 주어지면 $X$보다 작거나 같은 가장 큰 $K$-균형 잡힌 수를 구해보자.</p>

### 입력

<p>첫째 줄에 테스트 케이스의 개수 $T$가 주어진다. $(1 \leq T \leq 10^5)$</p>

<p>둘째 줄부터 $T$개의 줄에 각각 양의 정수&nbsp;$X, K$가 공백으로 구분되어 주어진다. ($1 \leq X &lt; 10^{10^5}$, $1 \leq K \leq 10^5$)</p>

<p>모든 테스트 케이스에서 $X$의 <strong>길이의 합</strong>은 $10^5$ 이하이다.</p>

### 출력

<p>각 테스트 케이스의 정답을 한 줄에 하나씩 차례대로 출력한다.</p>

### 힌트

<ul>
	<li>입력과 출력은 64비트 정수 범위를 넘을 수 있다.</li>
	<li>사용하지 않은 숫자는 등장 횟수를 고려하지 않는다.</li>
	<li>Python 사용자는 PyPy로 제출하는 것을 권장한다.</li>
</ul>
