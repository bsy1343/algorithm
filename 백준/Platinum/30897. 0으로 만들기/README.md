# [Platinum III] 0으로 만들기 - 30897

[문제 링크](https://www.acmicpc.net/problem/30897)

### 성능 요약

시간 제한: 1 초, 메모리 제한: 1024 MB

### 통계

제출: 90, 정답: 35, 맞힌 사람: 22, 정답 비율: 44.000%

### 분류

애드 혹, 수학, 비둘기집 원리, 누적 합

### 문제 설명

<p>$N$개의 정수로 이루어진 수열 $A_1, A_2, ..., A_N$이 주어집니다. 여러분은 수와 수 사이에 연산자를 하나씩 끼워 넣어서 수식을 완성해야 합니다. 단, 수열의 순서는 바꿀 수 없습니다.</p>

<p>연산자는 덧셈(+), 뺄셈(-), 곱셈(&times;)만 사용할 수 있으며, 연산자의 우선순위는 곱셈이 덧셈과 뺄셈보다 앞섭니다.</p>

<p>또한 여러분은 완성된 수식에 최대 하나의 괄호를 사용할 수 있는데, 여는 괄호 뒤에는 반드시 숫자가 등장해야 하며, 마찬가지로 닫는 괄호 앞에는 반드시 숫자가 등장해야 합니다. 사칙연산의 규칙상, 괄호 안에 들어있는 식을 먼저 계산해야 합니다.</p>

<p>다음은 올바른 수식과 그렇지 않은 수식의 예시입니다.</p>

<ul>
	<li>$-1+2\times 6+7+(2-4)\times 9$: 맨 앞의 숫자에 연산자가 사용되었으므로, 올바르지 않은 수식입니다.</li>
	<li>$2\times 6(+3-5)-2\times 8+1$: 여는 괄호 뒤에 연산자가 등장하였으므로, 올바르지 않은 수식입니다.</li>
	<li>$3\times 6\quad 2+5-9\times 0+1$: 숫자 사이에 연산자가 사용되지 않았으므로, 올바르지 않은 수식입니다.</li>
	<li>$2\times (3)-5+4\times 2+1-9\times 3$: 올바른 수식입니다.</li>
	<li>$2023\times 12-2\times (14-0)$: 올바른 수식입니다.</li>
	<li>$2023)\times 12-2\times (17-0$: 괄호의 순서가 바뀌었으므로, 올바르지 않은 수식입니다.</li>
	<li>$(12+34)\times (56\times 78)-90$: 두 쌍 이상의 괄호가 사용되었으므로, 올바르지 않은 수식입니다.</li>
</ul>

<p>연산자와 괄호를 적절히 추가하여 완성된 수식의 결괏값이 $0$이 되도록 할 수 있다면 첫째 줄에 <code><span style="color:#e74c3c;">YES</span></code>를 출력하고, 둘째 줄에 결괏값이 $0$이 되도록 하는 완성된 수식을 출력해 주세요.</p>

<p>만약 연산자와 괄호를 어떻게 사용해도 수식의 결괏값을 $0$으로 만들 수 없다면 <span style="color:#e74c3c;"><code>NO</code></span>를 출력해 주세요.</p>

### 입력

<p>첫째 줄에 수의 개수 $N$이 주어집니다. $(30,000&le;N&le;50,000)$</p>

<p>둘째 줄에는 수열 $A_1, A_2, ..., A_N$이 공백으로 구분되어 주어집니다. $(0&le;A_i&le;10,000)$</p>

### 출력

<p>연산자와 괄호를 적절히 추가하여 완성된 수식의 결괏값이 $0$이 되도록 할 수 있다면 첫째 줄에 <code><span style="color:#e74c3c;">YES</span></code>를 출력하고, 둘째 줄에 결괏값이 $0$이 되도록 하는 완성된 수식을 출력해 주세요. 단, 곱셈 기호는 <span style="color:#e74c3c;"><code>*</code></span>를 사용하여 출력해야 합니다.</p>

<p>만약 연산자와 괄호를 어떻게 사용해도 수식의 결괏값을 $0$으로 만들 수 없다면 <span style="color:#e74c3c;"><code>NO</code></span>를 출력해 주세요.</p>
