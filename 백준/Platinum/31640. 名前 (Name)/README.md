# [Platinum I] 名前 (Name) - 31640

[문제 링크](https://www.acmicpc.net/problem/31640)

### 성능 요약

시간 제한: 1 초, 메모리 제한: 1024 MB

### 통계

제출: 56, 정답: 13, 맞힌 사람: 13, 정답 비율: 43.333%

### 분류

다이나믹 프로그래밍, 그래프 이론, 문자열, 그래프 탐색, 집합과 맵, 너비 우선 탐색, 비트마스킹

### 문제 설명

<p>JOI 君と IOI 君は犬を飼うことにした．最初にすべきことは，犬の名前を決めることである．二人で話し合った結果，犬の名前を以下の <var>4</var> つの条件を満たすものにすることにした．</p>

<p><strong>条件 1</strong>　名前は英大文字 (<code>A</code>, <code>B</code>, ..., <code>Z</code>) と英小文字 (<code>a</code>, <code>b</code>, ..., <code>z</code>) からなる文字列である．</p>

<p><strong>条件 2</strong>　JOI 君の好きな文字列は長さ <var>N</var> の文字列 <var>S</var> であるから，<var>S</var> が名前の部分列となるようにする．</p>

<p><strong>条件 3</strong>　IOI 君の好きな文字列は長さ <var>M</var> の文字列 <var>T</var> であるから，<var>T</var> が名前の部分列となるようにする．</p>

<p><strong>条件 4</strong>　名前を呼びやすいものにするため，同じ文字の間には別の文字が <var>K</var> 文字以上あるようにする．厳密には，位置が異なる任意の同じ文字 <var>2</var> つについて，必ずその間に別の文字が <var>K</var> 文字以上入っているようにする．なお，<var>K = 0</var> の場合や，名前の文字がすべて異なる場合は，この条件を満たしていると考える．</p>

<p>ただし，これらの条件では英大文字と英小文字は区別される．例えば <code>A</code> と <code>a</code> は異なる文字とみなされることに注意せよ．</p>

<p>ここで，名前の部分列とは，名前から何文字か (<var>0</var> 文字でもよい) を取り除いて作れる文字列のことである．例えば，名前が <code>algorithm</code> であるとき，<code>ai</code> や <code>lgtm</code> は名前の部分列であるが，<code>joi</code> や <code>logarithm</code> は名前の部分列ではない．</p>

<p>二人は名前が短いほど良いと考えているため，<var>4</var> つの条件のもとで最も短い名前を付けることにした．</p>

<p>JOI 君と IOI 君の好きな文字列および整数 <var>K</var> が与えられたとき，犬に付ける名前の文字数を求めるプログラムを作成せよ．</p>

### 입력

<p>入力は以下の形式で与えられる．</p>

<pre>
<var>N</var>   <var>M</var>   <var>K</var>
<var>S</var>
<var>T</var></pre>

### 출력

<p>犬に付ける名前の文字数を <var>1</var> 行で出力せよ．</p>

### 제한

<ul>
	<li><var>1 ≦ N ≦ 500</var>．</li>
	<li><var>1 ≦ M ≦ 500</var>．</li>
	<li><var>0 ≦ K ≦ 3</var>．</li>
	<li><var>S</var> は英大文字と英小文字からなる長さ <var>N</var> の文字列である．</li>
	<li><var>T</var> は英大文字と英小文字からなる長さ <var>M</var> の文字列である．</li>
	<li><var>N, M, K</var> は整数である．</li>
</ul>
