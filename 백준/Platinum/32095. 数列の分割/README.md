# [Platinum II] 数列の分割 - 32095

[문제 링크](https://www.acmicpc.net/problem/32095)

### 성능 요약

시간 제한: 8 초 (추가 시간 없음), 메모리 제한: 1024 MB

### 통계

제출: 8, 정답: 3, 맞힌 사람: 3, 정답 비율: 60.000%

### 분류

다이나믹 프로그래밍, 자료 구조, 우선순위 큐

### 문제 설명

<p>長さ <i>n</i> の整数列 <i>A = (a<sub>1</sub>, ...,a<sub>n</sub>)</i> が与えられる．<i>m</i> 個の整数列 <i>B<sub>1</sub>, ..., B<sub>m</sub></i> (<i>m &ge; 1</i>) が以下の条件をともに満たすとき，<i>(B<sub>1</sub>, ..., B<sub>m</sub>)</i> を <i>A</i> の分割と呼ぶことにする．</p>

<ul>
	<li>いずれの <i>B<sub>i</sub></i> (<i>1 &le; i &le; m</i>) も，長さが 1 以上の整数列である．</li>
	<li><i>B<sub>1</sub>, ..., B<sub>m</sub></i> をこの順につなげたものは <i>A</i> に等しい．</li>
</ul>

<p>たとえば，((3, 1, 4, 1, 5)) や ((3), (1, 4), (1, 5)) などはいずれも (3, 1, 4, 1, 5) の分割である．</p>

<p>また，整数列 <i>B</i> に対し，<i>B</i> の要素の総和の 2 乗を <i>f(B)</i> とする．さらに，<i>A</i> の分割 <i>(B<sub>1</sub>, ..., B<sub>m</sub>)</i> に対し，<i>f(B<sub>1</sub>) + &hellip; + f(B<sub>m</sub>)</i> を，この分割のスコアと呼ぶことにする．たとえば，((3), (1, 4), (1, 5)) のスコアは <i>3<sup>2</sup> + (1+4)<sup>2</sup> + (1+5)<sup>2</sup> = 70</i> である．</p>

<p><i>A</i> の分割はちょうど <i>2<sup>n-1</sup></i> 個存在する．<i>A</i> のすべての分割に対するスコアを降順に並べたとき，<i>k</i> 番目となる値を求めよ．</p>

<p>たとえば，Sample Input の 2 つ目のデータセットでは，<i>A</i> の分割のスコアを降順に並べると 196, 130, 116, 110, 106, 100, 90, 70, 70, 68, 66, 62, 60, 60, 58, 52 となるため，6 番目の値は 100 である．</p>

### 입력

<p>入力は複数のデータセットからなる．データセットの個数は 30 を超えない．各データセットは次の形式で表される．</p>

<blockquote>
<p><i>n</i> <i>k</i></p>

<p><i>a<sub>1</sub></i> <i>a<sub>2</sub></i> <i>&hellip;</i> <i>a<sub>n</sub></i></p>
</blockquote>

<p><i>n</i> および <i>k</i> はいずれも整数であり，<i>1 &le; n &le; 1000</i> および <i>1 &le; k &le; min{2<sup>n-1</sup>, 2000}</i> を満たす．各 <i>a<sub>i</sub></i> は数列 <i>A</i> の <i>i</i> 番目の要素となる整数であり，<i>-10<sup>6</sup> &le; a<sub>i</sub> &le; 10<sup>6</sup></i> を満たす．</p>

<p>入力の終わりは 2 つのゼロからなる行で表される．</p>

### 출력

<p>各データセットについて，<i>A</i> のすべての分割のスコアのうち <i>k</i> 番目に大きい値を，1 行に出力せよ．</p>
