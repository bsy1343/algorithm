### 문제 설명

<p>한신이는 대학교에서&nbsp;&quot;복붙의 달인&quot;으로 유명하다. 한신이는&nbsp;타이핑 속도가 느리기 때문에&nbsp;대학에서 가능한 모든 일을 복붙으로 해결한다.&nbsp;그는 n개의 문자를 입력하는데 있어서 n초의 시간이 걸리지만&nbsp;뛰어난&nbsp;&quot;붙여넣기&quot; 스킬을 이용하면&nbsp;어떠한 개수의 문자도 단 1초만에 타이핑 할 수 있다. 만약 한신이가 &quot;bana&quot;를 복사한 상태에서&nbsp;&quot;banana&quot;를 타이핑한다면, &quot;bana&quot; 붙여넣기 1초, &#39;n&#39; 입력, &#39;a&#39; 입력으로 총 3초가 걸린다.&nbsp;한신이가 클립보드에 저장한&nbsp;p를 알고 있을 때 s를 입력하는데 걸리는 최소 시간을 계산해보자!</p>

### 입력

<p>첫 번째 줄에는 테스트케이스의 개수 T(T &le; 25)가 입력된다. 각 테스트케이스는 한 줄에 2개의 문자열 s와 p가&nbsp;공백으로 구분되어 입력되며&nbsp;한신이는 p를 복사하여 s를 만들어 내는&nbsp;것을 목표로 한다.&nbsp;s의 최대 길이는 10,000이고, p의 최대 길이는 100이다.</p>

### 출력

<p>각&nbsp;테스트 케이스에 맞는 한신이가 p를 이용하여 s를 타이핑할 때 걸리는 최소 시간(초&nbsp;단위)을 출력하라!</p>
