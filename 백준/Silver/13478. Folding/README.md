### 문제 설명

<p>As you can remember, Alex is fond of origami. She switched from squares to rectangles, and rectangles&nbsp;are much more difficult to master. Her main interest is to determine what is the minimum possible&nbsp;number of folds required to transform W &times; H rectangle to w &times; h one. The result of each fold should also&nbsp;be rectangular, so it is only allowed to make folds that are parallel to the sides of the rectangle.</p>

<p>Help Alex and write a program that determines the minimum required number of folds.</p>

### 입력

<p>The first line of the input contains two integers W and H &mdash; the initial rectangle dimensions. The second&nbsp;line contains two more integers w and h &mdash; the target rectangle dimensions (1 &le; W, H, w, h &le; 10<sup>9</sup>).</p>

### 출력

<p>Output a single integer &mdash; the minimum required number of folds to transform the initial rectangle to&nbsp;the target one.</p>

<p>If the required transformation is not possible, output &minus;1.</p>

### 힌트

<p>In the first example you should fold 2 &times; 7 rectangle to 2 &times; 4, and then to 2 &times; 2.</p>
